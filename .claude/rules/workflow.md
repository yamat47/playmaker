# 実装フロー規約

実装ユニットが一段落したら、以下を **この順で必ず** 実行する。トリガーは「コード変更があり、論理的な区切りに達した」状態（＝判断。1 ファイルの途中編集や説明だけのターンは対象外）。

## 1. /simplify を無確認で実行する

実装が一段落したと判断したら、ユーザーに尋ねず `/simplify` を実行する。レビュー指摘のうち **changed コード内・低リスク・規約準拠** のものを反映し、誤検出/スコープ外は理由を添えて見送る。反映後はローカル CI（typecheck / lint / test+coverage / build）が緑であることを確認する。

## 2. /create-pr の実行可否をユーザーに質問する

`/simplify` 完了後、コミット / PR 作成は **ユーザー判断**とする。`/create-pr` を実行してよいか **必ずユーザーに質問**し、明示の承認があるまでコミット・push・PR 作成は行わない。

## 適用範囲・例外

- 「一段落」は実装の論理的区切り（機能追加・リファクタ・バグ修正の完了）。途中経過の報告ターンでは発火させない
- ユーザーが明示的に「simplify 不要」「すぐ PR」等と指示した場合はその指示を優先する
- ドキュメント / 設定のみの軽微変更で `/simplify` が無意味な場合は、その旨を述べて 2 へ進んでよい
